# 未成年人电子产品使用风险评估的视频文本化前置组件方案

## 1. 背景与目标

当前数据集包含三类信息：

- 未成年人使用电子产品的视频，单段约 30 分钟。
- 未成年人填写的电子产品使用调查问卷。
- 未成年人心率、表情等辅助信息。

现有标签由人工综合问卷、心率、表情和视频行为等信息得到，例如无明显风险、轻度风险、重度风险等。

目标系统在部署阶段只接收视频输入，输出风险判断结果。由于 LLM 是最终判断链路中的关键组件，但不应直接接触未成年人原始视频、音频或图像，因此需要一个前置组件，将原始视频转换为 LLM 可接受的纯文本证据。

本方案将该前置组件定义为：

> 面向未成年人电子产品使用风险评估的隐私保护视频到文本证据编码器。

该组件的目标不是简单地将视频 caption 成自然语言，而是在隐私保护约束下，从视频中提取与风险判断相关的可观察代理特征，并生成结构化、可审计、可复现的文本证据。

## 2. 核心贡献定位

本项目可以形成三个主要贡献。

### 2.1 前置组件

提出一个多管线的隐私保护视频文本化前置组件。该组件由关键时间窗提取、关键区域聚焦、参考序列特征提取、特征归一化、质量评估和隐私文本生成模块组成。

组件输入为原始视频，输出为纯文本证据包。LLM 只接收文本，不接收原始视频、图像、音频或高维生物特征。

### 2.2 文本化数据集

将已采集的视频经过前置组件处理，生成模板化、结构化、隐私过滤后的文本证据，并与原有风险标签一起组成可发布的数据集。

该数据集不公开原始视频、图片、音频、ASR 原文、OCR 原文或可重识别的高维特征，而是公开经过粒度压缩和隐私审查后的行为证据文本。

### 2.3 Text-only LLM 评测基准

基于上述文本化数据集，建立 text-only LLM 风险分级评测基准，比较不同 LLM、传统文本分类器、时序模型以及不同前置组件配置的表现。

建议将最终项目定位为：

```text
Privacy-preserving Video-to-Text Evidence Encoding
for Minor Digital Device Use Risk Screening
```

即：面向未成年人电子产品使用风险筛查的隐私保护视频文本证据编码。

## 3. 关键原则

### 3.1 视频只能提供代理证据

原始标签来自问卷、心率、表情和人工综合判断，但部署阶段只有视频输入。因此前置组件不能声称从视频中完整还原问卷或真实心理状态，只能提取可观察代理特征。

可观察代理特征包括：

- 屏幕注视持续性。
- 手部操作频率和重复操作。
- 姿态变化。
- 面部动作单元。
- 情绪趋势。
- 疑似挫败、烦躁、疲劳、过度沉浸等行为线索。
- 可选的 rPPG 心率趋势，但必须带质量评分。

系统输出应被定义为风险筛查结果，而不是医学诊断或心理诊断。

### 3.2 纯文本不等于天然匿名

即使输出不包含图像，文本中仍可能包含敏感信息，例如：

- 未成年人行为模式。
- 表情和情绪趋势。
- 心率代理特征。
- 使用风险标签。
- 罕见行为事件。
- 可反推出身份的场景或屏幕内容。

因此文本数据集仍应按敏感派生数据管理，公开前必须经过隐私审查。

### 3.3 不使用自由 caption 作为主要输出

不建议使用视觉语言模型直接生成自由描述，例如：

```text
一个穿蓝色校服的小男孩坐在卧室里玩手机，看起来很焦虑。
```

这类自由文本存在多重问题：

- 可能包含校服、房间、性别、年龄、外貌等身份线索。
- 可能引入未经验证的心理状态推断。
- 不同模型生成风格不稳定。
- 难以复现和审计。

推荐路径是：

```text
结构化视觉特征 -> 隐私过滤 -> 固定模板文本 -> LLM
```

而不是：

```text
视频 -> 自由 caption -> LLM
```

## 4. 总体架构

整体架构如下：

```text
Raw Video
  -> Authorization and Isolated Storage
  -> Key-window Extractor
  -> Region-of-Interest Focuser
  -> Reference-sequence Feature Extractor
  -> Feature Normalizer and Quality Estimator
  -> Privacy and Text Sanitizer
  -> Structured Text Evidence
  -> Text-only LLM
  -> Risk Level, Evidence, Confidence, Review Flag
```

各模块职责如下：

| 模块 | 输入 | 输出 | 作用 |
| --- | --- | --- | --- |
| 授权与隔离存储 | 原始视频 | 匿名样本 ID、受控视频副本 | 管理授权、访问控制、保存期限 |
| 关键时间窗提取器 | 原始视频或低级特征 | 关键窗口列表 | 选择最有信息量的视频片段 |
| 关键区域聚焦器 | 视频帧 | 人脸、手部、设备、屏幕、背景敏感区域 | 只分析必要区域，降低隐私暴露 |
| 参考序列特征提取器 | 全视频或基线窗口 | 个体基线、群体参考、窗口差异 | 做个体内归一化 |
| 特征归一化与质量评估器 | 原始视觉特征 | 归一化特征、质量分 | 避免低质量特征误导 LLM |
| 隐私与文本生成器 | 结构化特征 | JSON 和模板文本 | 生成 LLM 可读文本证据 |
| Text-only LLM | 文本证据 | 风险判断结果 | 综合证据、解释、输出置信度 |

## 5. 模块设计

## 5.1 关键时间窗提取器

该模块不应只提取单张关键帧，而应提取关键时间窗。原因是很多风险线索不是静态帧可见的，而是短时间内的变化过程。

建议使用多策略采样：

### 覆盖性采样

保证 30 分钟视频全程都有代表，例如每 30 秒或 60 秒生成一个基础窗口摘要。

### 事件性采样

捕捉具有判断价值的突变事件，例如：

- 操作频率突然升高。
- 连续重复点击。
- 面部负向动作单元上升。
- 姿态明显前倾或僵硬。
- 长时间持续注视屏幕。
- 眨眼频率显著下降。
- 被打断、暂停或设备移开后的反应。
- 疑似挫败、烦躁、抗拒或疲劳片段。

### 质量性采样

优先选择人脸、手部、设备区域可见度较高、光照较好、遮挡较少的片段。

### 平衡采样

不要只抽取异常片段，也要保留平稳基线片段。否则 LLM 可能因为只看到异常证据而高估风险。

关键时间窗提取器的输出示例：

```json
{
  "selected_windows": [
    {
      "start_sec": 0,
      "end_sec": 30,
      "reason": "baseline_sampling",
      "quality": "high"
    },
    {
      "start_sec": 720,
      "end_sec": 750,
      "reason": "interaction_spike",
      "quality": "medium"
    },
    {
      "start_sec": 1260,
      "end_sec": 1290,
      "reason": "negative_affect_peak",
      "quality": "high"
    }
  ]
}
```

## 5.2 关键区域聚焦器

关键区域聚焦器负责识别视频中的必要区域和敏感区域。它不是为了把裁剪图片交给 LLM，而是为了让本地 CV 模型只分析必要区域，并过滤可能泄露身份的信息。

建议识别以下区域：

- 主体儿童区域。
- 面部区域。
- 眼睛和头部方向区域。
- 手部区域。
- 电子设备区域。
- 屏幕区域。
- 背景敏感区域。
- 其他人区域。

### 主体儿童区域

用于提取：

- 坐姿和身体姿态。
- 低头、前倾、后仰。
- 身体僵硬或频繁扭动。
- 离开座位或靠近屏幕。

不应输出：

- 外貌描述。
- 服装颜色。
- 身材细节。
- 可识别身份的场景描述。

### 面部区域

用于提取：

- 面部动作单元 AUs。
- 眨眼频率。
- 头部姿态。
- 表情趋势。
- 可选 rPPG 心率趋势。

不应输出：

- 原始人脸图像。
- face embedding。
- 长相描述。
- 肤色描述。
- 五官描述。
- 可重识别的面部几何特征。

### 手部区域

用于提取：

- 点击频率。
- 滑动频率。
- 重复操作。
- 急促操作。
- 拿起或放下设备。
- 设备被移开后的反应。

### 设备和屏幕区域

建议只输出屏幕内容的高层类别，例如：

```text
game / short_video / chat / learning / browser / unknown
```

不应输出：

- 聊天内容。
- 用户名。
- 联系人。
- 账号。
- 具体视频标题。
- 学校平台名称。
- 定位、电话、班级群等文字。

### 背景区域

背景区域主要用于隐私检测，不应作为风险判断依据。

需要检测并过滤：

- 其他人脸。
- 家庭照片。
- 门牌。
- 证件。
- 校徽。
- 奖状。
- 地理位置。
- 家庭成员信息。
- 学校、班级或机构信息。

## 5.3 参考序列的帧特征提取器

参考序列用于建立个体内基线，减少个体差异对判断的干扰。

不同孩子在自然状态下的眨眼频率、表情幅度、姿态、操作速度、心率水平差异很大。如果只使用绝对值，模型可能把个体差异误判为风险信号。

可使用的参考序列包括：

- 前 3 到 5 分钟相对稳定片段。
- 低交互片段。
- 全视频中位数。
- 同年龄组群体参考，但需谨慎使用。

建议输出相对变化而非单纯绝对值：

```json
{
  "baseline": {
    "blink_rate": "normal",
    "head_forward_angle": "medium",
    "touch_rate": "low",
    "negative_affect": "low"
  },
  "current_window_delta": {
    "blink_rate": "-35%",
    "touch_rate": "+120%",
    "head_forward_angle": "+18deg",
    "negative_affect": "+2.1_zscore"
  }
}
```

同时保留少量与风险直接相关的绝对信息，例如：

- 最大连续注视屏幕时长。
- 全程离屏次数。
- 总有效操作时长。
- 30 分钟内高强度交互片段占比。

## 5.4 特征提取内容

建议提取以下几类特征。

### 注意力与沉浸相关特征

- 屏幕注视比例。
- 最大连续屏幕注视时长。
- 离屏次数。
- 离屏后回到屏幕的平均时间。
- 长时间低眨眼片段。
- 长时间固定姿态片段。

### 交互行为特征

- 点击频率。
- 滑动频率。
- 重复操作比例。
- 操作强度趋势。
- 操作峰值次数。
- 操作中断后恢复速度。

### 表情与情绪趋势特征

- AU4 皱眉相关动作。
- AU12 微笑相关动作。
- 眼睑、嘴角、眉部变化。
- 负向表情片段比例。
- 表情波动幅度。
- 疑似挫败、烦躁、疲劳片段。

### 姿态和身体状态特征

- 头部前倾角度。
- 身体前倾时长。
- 姿态僵硬程度。
- 频繁扭动或焦躁动作。
- 与屏幕距离变化。

### 可选 rPPG 特征

rPPG 可作为实验性模块，但不能作为默认可靠心率来源。

建议只输出带质量门控的趋势：

```json
{
  "rppg": {
    "usable_ratio": "40-60%",
    "estimated_hr_trend": "rising",
    "confidence": "low",
    "invalid_reasons": [
      "motion_artifact",
      "low_light"
    ]
  }
}
```

不建议在公开数据集中发布精确心率或 HRV 数值。

## 5.5 质量评估器

质量评估器必须与特征一起输出。没有质量信息，LLM 容易把低质量特征当作可靠证据。

建议包含：

- 人脸可见比例。
- 手部可见比例。
- 设备可见比例。
- 屏幕可见比例。
- 光照质量。
- 遮挡程度。
- 多人干扰程度。
- rPPG 可用比例。
- 有效观察时长。

质量输出示例：

```json
{
  "quality": {
    "overall": "medium",
    "face_visible_ratio_bin": "60-80%",
    "hands_visible_ratio_bin": "60-80%",
    "device_visible_ratio_bin": "80-100%",
    "lighting": "medium",
    "occlusion": "low",
    "multi_person_interference": "none"
  }
}
```

## 5.6 隐私与文本生成器

该模块负责将结构化特征转换为可供 LLM 使用的文本，同时清除或压缩敏感信息。

建议做以下处理：

- 删除视频元数据、文件名、设备 ID、GPS 等。
- OCR 检测并过滤屏幕和背景中的 PII。
- ASR 仅用于隐私检测或声音事件检测，不公开原始转写。
- 所有时间戳改为时间段。
- 精确坐标改为姿态类别。
- 精确心率改为趋势或区间。
- 精确 AU 数值改为上升、下降、稳定或等级。
- 不输出自由外貌描述。
- 不输出家庭、学校、地点等场景细节。

## 6. LLM 输入文本格式

建议同时支持 JSON 和模板化自然语言两种格式。

### 6.1 JSON 格式

JSON 适合模型输入、程序评测和数据集发布。

```json
{
  "sample_id": "anon_000127",
  "input_type": "video_only",
  "duration_bin": "25-30min",
  "quality": {
    "overall": "medium",
    "face_visible_ratio_bin": "60-80%",
    "hands_visible_ratio_bin": "60-80%",
    "device_visible_ratio_bin": "80-100%"
  },
  "global_features": {
    "screen_attention": "high",
    "max_continuous_attention": "8-12min",
    "interaction_intensity": "medium_high",
    "interaction_trend": "increasing_late",
    "negative_affect": "medium",
    "posture_risk": "medium_high"
  },
  "events": [
    {
      "time_bin": "08-10min",
      "event_type": "interaction_spike_with_frustration_cues",
      "evidence": [
        "touch_rate_above_baseline",
        "repeated_actions_increased",
        "AU4_increased"
      ],
      "confidence": 0.72
    },
    {
      "time_bin": "17-19min",
      "event_type": "sustained_attention_with_low_blink_rate",
      "evidence": [
        "long_screen_attention",
        "blink_rate_below_baseline",
        "forward_head_posture"
      ],
      "confidence": 0.81
    }
  ],
  "limitations": [
    "no_questionnaire",
    "no_long_term_usage_record",
    "no_contact_heart_rate",
    "video_only_screening"
  ]
}
```

### 6.2 模板化文本格式

模板化文本适合直接输入给纯文本 LLM。

```text
样本编号: anon_000127
输入类型: video_only
有效观察时长: 25-30分钟
数据质量: 中等
限制: 无问卷、无长期使用记录、无接触式心率数据

全局行为摘要:
- 屏幕注视比例: 高
- 最大连续屏幕注视时长: 8-12分钟
- 手部交互强度: 中高
- 操作强度趋势: 后半段升高
- 负向表情趋势: 中等
- 姿态风险: 中等偏高

关键事件:
1. 时间段 08-10分钟: 手部操作频率明显升高，重复点击增加，面部 AU4 上升，推测为挫败相关片段，置信度 0.72。
2. 时间段 17-19分钟: 出现持续屏幕注视、低于个体基线的眨眼频率和前倾姿态，置信度 0.81。

不可判断信息:
- 无法判断日常总使用时长
- 无法判断学习、家庭或社交功能损害
- 无法判断主观渴求或控制失败
- 无法替代临床或心理评估
```

## 7. LLM 判断模块设计

LLM 的角色不是直接诊断，而是基于文本证据进行风险分级、解释、置信度估计和复核建议。

推荐输出类别：

- no_observed_risk
- mild_risk
- moderate_risk
- high_risk
- insufficient_evidence

不建议使用：

- no_addiction
- mild_addiction
- severe_addiction

原因是“成瘾”具有医学和污名化含义，而单段视频更适合作为风险筛查。

### 7.1 Prompt 约束

建议在 system prompt 中加入以下约束：

```text
你只能基于输入文本证据进行判断。
不得根据性别、外貌、家庭环境、设备品牌、房间条件或其他身份相关信息推断风险。
不得输出医学诊断。
如果证据不足，必须输出 insufficient_evidence。
必须列出支持证据和缺失信息。
高风险或低置信度样本必须建议人工复核。
```

### 7.2 输出 schema

```json
{
  "risk_level": "mild_risk",
  "confidence": 0.68,
  "supporting_evidence": [
    "screen_attention_high",
    "interaction_intensity_increased_late",
    "frustration_related_event_detected"
  ],
  "missing_information": [
    "no_questionnaire",
    "no_long_term_usage_record",
    "no_contact_heart_rate"
  ],
  "needs_human_review": true
}
```

## 8. 文本化数据集发布策略

由于数据对象是未成年人，即使只公开文本也需要分级发布。

## 8.1 Public Lite 公开版

适合公开发布到论文、GitHub、HuggingFace 或开放数据平台。

可以包含：

- 匿名样本 ID。
- 风险标签。
- train、validation、test split。
- 视频时长区间。
- 数据质量等级。
- 聚合后的行为特征。
- 粗粒度时间窗口。
- 事件类别。
- 事件置信度。
- 不可判断信息。
- 固定模板文本。

不应包含：

- 原始视频。
- 图片。
- 音频。
- ASR 原文。
- OCR 原文。
- 精确心率或 HRV。
- 精确坐标。
- face embedding。
- 高维骨架序列。
- 地域、学校、家庭描述。
- 自由 caption。

公开版样例：

```json
{
  "sample_id": "anon_pub_0421",
  "label": "mild_risk",
  "duration_bin": "25-30min",
  "quality": "medium",
  "global_text": "该样本在有效观察时长内表现出高屏幕注视比例、中高操作强度和后半段交互增强趋势。",
  "event_texts": [
    "08-10分钟出现操作强度升高，并伴随挫败相关面部动作增加。",
    "17-19分钟出现持续前倾和低眨眼频率，提示高专注或疲劳相关状态。"
  ],
  "limitations": [
    "video_only",
    "no_questionnaire",
    "no_long_term_record"
  ]
}
```

## 8.2 Controlled Research 受控研究版

只提供给签署数据使用协议的研究者。

可以包含：

- 更细粒度窗口特征。
- 更详细事件序列。
- 相对归一化数值。
- 更多质量指标。

仍不应包含：

- 原始视频。
- 原始图像。
- 原始音频。
- ASR 原文。
- OCR 原文。
- 可重识别高维生物特征。

访问条件建议包括：

- 签署数据使用协议。
- 禁止重识别。
- 禁止再分发。
- 禁止用于个体诊断、惩戒、画像或商业决策。
- 到期删除。
- 下载审计。

## 8.3 Raw/Internal 内部版

仅限机构内部使用。

可包含：

- 原始视频。
- 原始音频。
- 原始 OCR 和 ASR。
- 中间图像。
- 高维特征。
- 标注记录。

该版本不公开。

## 9. 公开前隐私审查

公开数据集前建议设置隐私发布门禁。

### 9.1 PII 扫描

检查是否包含：

- 人名。
- 学校。
- 班级。
- 地点。
- 电话。
- 账号。
- App 用户名。
- 家庭成员称谓。
- 聊天内容。

### 9.2 稀有组合检查

评估以下组合是否可能重识别个体：

- 标签 + 年龄段 + 场景 + 罕见事件。
- 时间段 + 特定行为 + 屏幕类别。
- 质量特征 + 行为特征 + 数据来源。

### 9.3 粒度压缩

建议将高风险字段压缩为粗粒度形式：

| 原始字段 | 公开形式 |
| --- | --- |
| 精确时间戳 | 时间段 |
| 精确心率 | 趋势或区间 |
| 精确 AU 数值 | 上升、下降、稳定、等级 |
| 精确坐标 | 姿态类别 |
| 高频序列 | 聚合统计 |
| 屏幕 OCR | 语义类别 |
| ASR 原文 | 非身份化声音事件 |

### 9.4 人工抽检

建议对以下样本重点检查：

- 高风险标签样本。
- 罕见行为样本。
- 文本特别长的样本。
- 包含声音事件或屏幕事件的样本。
- 自动 PII 检测置信度低的样本。

### 9.5 攻击评估

模拟攻击者具有部分背景知识的场景：

- 攻击者知道某个孩子参加过采集。
- 攻击者知道采集地点或大致时间。
- 攻击者知道孩子使用某类 App。
- 攻击者知道孩子有某种罕见行为表现。

评估公开文本是否可能帮助攻击者锁定样本。

## 10. 实验设计

为了证明前置组件的有效性，建议做以下实验。

### 10.1 模型比较

- 传统文本分类器。
- 小型开源 LLM。
- 大型闭源 LLM。
- 时序模型。
- 时序模型 + LLM。

### 10.2 消融实验

建议比较：

- 只用全局统计文本。
- 全局统计 + 固定时间窗文本。
- 全局统计 + 关键事件文本。
- 全局统计 + 固定时间窗 + 关键事件。
- 不使用参考序列归一化。
- 使用参考序列归一化。
- 不使用 rPPG。
- 使用 rPPG，但带质量门控。
- 自由 caption 文本。
- schema-first 模板文本。

预期结论可能包括：

- schema-first 模板文本比自由 caption 更稳定。
- 关键事件比单纯均匀摘要更有用。
- 参考序列归一化提升泛化能力。
- rPPG 只在高质量视频中可能有增益。
- 质量报告有助于 LLM 输出 insufficient_evidence。

### 10.3 隐私与性能权衡

比较不同隐私粒度下的模型性能：

- 精确数值。
- 区间数值。
- 趋势描述。
- 仅事件类别。

评估隐私压缩对准确率、宏平均 F1、召回率和校准误差的影响。

### 10.4 视频-only 上限评估

建议比较三类输入：

- 完整数据：问卷 + 心率 + 表情 + 视频。
- 视频-only 结构化特征。
- 视频-only 文本证据 + LLM。

该实验用于量化视频-only 任务与完整标注依据之间的信息差。

如果视频-only 性能明显低于完整数据，这是合理结果，说明部署输入本身存在信息上限，而不是前置组件失败。

## 11. 评价指标

建议使用以下指标：

### 分类性能

- Accuracy。
- Macro F1。
- Weighted F1。
- Per-class recall。
- High-risk recall。
- Confusion matrix。

### 可靠性

- Calibration error。
- 低置信度样本比例。
- insufficient_evidence 触发率。
- 人工复核命中率。

### 隐私保护

- PII 泄露率。
- OCR/ASR 残留敏感信息比例。
- 重识别攻击成功率。
- 稀有组合风险样本比例。

### 文本质量

- schema 合法率。
- 字段缺失率。
- 事件覆盖率。
- 模板一致性。
- 可解释证据完整率。

## 12. 风险与边界

### 12.1 rPPG 风险

rPPG 对光照、遮挡、运动、肤色、帧率、视频压缩、面部角度都很敏感。它应被视为可选代理特征，而不是可靠的接触式心率替代。

公开数据集中不建议发布精确心率或 HRV。

### 12.2 心理状态推断风险

系统不应直接输出：

- 这个孩子焦虑。
- 这个孩子成瘾。
- 这个孩子自控力差。
- 这个孩子有心理疾病。

更合适的输出是：

- 观察到负向表情相关线索增加。
- 观察到操作强度升高。
- 观察到长时间持续注视屏幕。
- 观察到风险相关代理特征，但缺少问卷和长期行为信息。

### 12.3 标签污名化风险

建议使用风险分级标签，而不是成瘾诊断标签：

```text
no_observed_risk
mild_risk
moderate_risk
high_risk
insufficient_evidence
```

避免使用：

```text
no_addiction
mild_addiction
severe_addiction
```

### 12.4 单段视频的信息上限

很多成瘾或使用障碍相关标准涉及长期行为模式、控制失败、负面后果和功能损害。单段 30 分钟视频无法完整判断这些信息。

因此系统应明确定位为辅助筛查，不能替代问卷、临床访谈、心理评估或监护人/教师长期观察。

## 13. 合规和伦理要求

由于对象是未成年人，且数据可能包含视频、声音、面部、生理代理特征和风险标签，应按高敏感数据处理。

建议遵守以下原则：

- 明确监护人同意。
- 明确数据用途和保存期限。
- 原始视频隔离保存。
- 最小必要处理。
- 不将原始视频提供给公有 LLM。
- 不公开原始视频、图像或音频。
- 日志审计和访问控制。
- 到期删除。
- 数据集仅用于研究，不用于个体诊断、惩戒、画像或商业决策。

公开数据集 README 中应明确写明：

```text
本数据集仅用于群体层面的算法研究和模型评测。
本数据集不应用于对单个未成年人进行自动化诊断、惩戒、画像、教育管理或商业决策。
标签为研究场景下的风险分级，不构成医学或心理诊断。
```

## 14. 推荐落地路线

### 第一阶段：MVP

- 只做 text-only 路线。
- 提取姿态、手部、头部方向、AUs、眨眼、交互强度。
- 不启用或仅实验性启用 rPPG。
- 生成 JSON 和模板化文本。
- 使用纯文本 LLM 做初步风险分级。
- 高风险和低置信度样本进入人工复核。

### 第二阶段：数据集构建

- 对历史视频批量生成文本证据。
- 做 PII 扫描、粒度压缩、人工抽检。
- 划分 public lite、controlled research 和 internal raw 三个版本。
- 编写数据集说明、标签说明、限制说明和使用协议。

### 第三阶段：评测基准

- 比较不同 LLM 和传统模型。
- 做消融实验。
- 做隐私-性能权衡实验。
- 做视频-only 上限评估。

### 第四阶段：增强版

- 引入更强的关键事件检测。
- 引入时序模型辅助评分。
- 在充分验证后加入 rPPG 趋势。
- 增加人工复核闭环。
- 持续更新数据治理和隐私审查流程。

## 15. 总结

该方案的核心不是让 LLM 看懂原始视频，而是在隐私保护约束下，把视频中可用于风险筛查的行为证据转化为结构化纯文本。

推荐最终系统定义为：

```text
多管线隐私保护视频到文本证据编码器
```

其关键模块包括：

- 关键时间窗提取器。
- 关键区域聚焦器。
- 参考序列特征提取器。
- 特征归一化与质量评估器。
- 隐私过滤与模板化文本生成器。

其主要产出包括：

- 一个可复现的前置组件。
- 一个不公开原始视频的文本化行为证据数据集。
- 一个 text-only LLM 风险分级评测基准。

该定位比“视频转文本”更准确，也比“LLM 直接判断成瘾”更稳健。它明确承认视频-only 输入的信息上限，同时通过结构化证据、隐私保护、质量评估和人工复核机制提升系统的可信度和可用性。
